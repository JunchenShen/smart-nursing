# 模拟数据生成的客观规律说明

本项目当前没有接入真实护理培训机构数据，因此采用程序生成模拟数据。生成逻辑不是简单随机填充，而是围绕护理培训业务中的“学员能力、课程难度、学习投入、资源完成度、考核成绩”之间的关系建立可解释规则。

## 生成目标

模拟数据需要满足三点：

- 结构真实：数据表覆盖学员、课程、资源、学习行为、考核成绩。
- 关系合理：学习投入越充分，学习进度和成绩整体越高；课程越难，平均成绩相对越低。
- 可复现：使用固定随机种子，便于实验复现和答辩演示。

## 主要业务假设

### 0. 学员姓名采用三字不重复组合

学员姓名采用“姓 + 名首字 + 名尾字”的三字结构。程序内置：

```text
50 个姓氏 × 30 个名首字 × 30 个名尾字 = 45000 个姓名组合
```

默认按学员序号稳定映射姓名，因此在 45000 人以内可以保证姓名不重复。课程项目推荐的 3000-5000 人规模不会出现重名问题，同时姓名形式比简单两字名更接近真实培训名单。

### 1. 学员基础能力服从近似正态分布

程序为每个学员生成一个隐藏变量 `base_ability`，默认均值为 74、标准差为 10。

这符合培训场景中的常见规律：大部分学员能力集中在中间水平，少数学员显著更强或更弱。该变量不会直接展示给用户，但会影响期中成绩、期末成绩和实操成绩。

对应代码参数：

```text
BASE_ABILITY_MEAN = 74
BASE_ABILITY_STDDEV = 10
```

### 2. 课程难度对成绩产生负向影响

课程分为初级、中级、高级。难度越高，对成绩的惩罚越大：

| 难度 | 成绩惩罚 |
| --- | --- |
| 初级 | 0 |
| 中级 | 5 |
| 高级 | 9 |

这样可以解释为什么高级课程，例如康复训练辅助实操、临终关怀与心理支持，平均成绩可能低于基础照护课程。

### 3. 学习投入对成绩产生正向影响

学习投入由两个因素共同表示：

- 学习时长 `study_minutes`
- 完成资源数量 `completed_count`

程序设置了 `engagement_bonus`，学习时长越多、完成资源越多，成绩加成越高，但设置了上限，避免出现无限刷时长导致成绩不合理升高。

简化公式：

```text
engagement_bonus = min(12, study_minutes / course_hours / 6) + completed_count * 1.2
```

### 4. 学习进度影响期末成绩

期末成绩由期中成绩、平均学习进度、基础能力共同影响：

```text
final_score = midterm * 0.42 + avg_progress * 0.28 + base_ability * 0.30 + random_noise
```

这个规则表达了一个培训评价逻辑：最终成绩既看阶段考试，也看过程学习行为，同时保留少量随机波动，模拟考试状态、题目差异、主观评分等现实因素。

### 5. 移动端学习占比更高

老年护理培训机构中，移动端适合碎片化学习和课后复习，因此程序设置移动端占比较高：

```text
MOBILE_PLATFORM_WEIGHT = 0.64
MOBILE_EVENT_WEIGHT = 0.67
```

### 6. 资源访问不是 100%

并非每个学员都会访问每个课程资源。程序设置资源访问概率为 0.84，用来模拟漏学、跳学、只看重点资源等行为。

```text
RESOURCE_ACCESS_PROBABILITY = 0.84
```

## 如何扩大数据规模

默认生成 120 名学员，适合快速演示。可以通过命令行参数扩大规模：

```bash
python scripts/generate_sample_data.py --students 1000 --event-days 180
```

生成更大数据后，学习行为日志和成绩记录会随学员数同步扩大。大致关系如下：

```text
学习行为记录数 ≈ 学员数 × 每人选课数 × 每课资源数 × 资源访问概率
成绩记录数 ≈ 学员数 × 每人选课数
```

例如 1000 名学员、每人约 5.5 门课、每课 5 个资源、资源访问概率 0.84，则学习行为记录约为：

```text
1000 × 5.5 × 5 × 0.84 ≈ 23100 条
```

## 为什么这种模拟方式可解释

该模拟数据不是纯随机，而是由业务规则驱动：

- 学员成绩受基础能力、课程难度、学习投入共同影响。
- 学习行为日志与最终成绩之间存在正相关关系。
- 课程难度、课程标签和资源格式会造成可观察的差异。
- 高风险学员可以通过“低成绩、低进度、低学习时长”识别。

因此，在没有真实机构数据授权的情况下，这套数据可以用于开发、测试、答辩演示和大数据分析流程验证。
