# 登录与角色权限说明

系统内置了轻量级登录与角色权限，用于课程设计演示和答辩说明。当前版本没有接入独立用户中心，而是通过 Flask Session 和内置演示账户实现。

## 演示账户

| 角色 | 账号 | 默认密码 | 可见范围 |
| --- | --- | --- | --- |
| 管理员 | `admin` | `admin123` | 全局指标、全部课程、全部风险学员 |
| 培训教师 | `teacher` | `teacher123` | 负责护理标签下的课程和风险学员 |
| 机构负责人 | `org` | `org123` | 本机构高风险学员和培训建议 |

## 权限设计

### 管理员

管理员用于培训平台运营视角，可以查看全局指标：

- 总体培训效果
- 全部课程分析
- 全部薄弱标签
- 全部高风险学员
- 资源格式与学习趋势

### 培训教师

培训教师用于课程教学视角，默认负责以下护理标签：

```text
压疮预防、跌倒预防、感染防控
```

教师登录后，后端会过滤课程效果、薄弱标签和高风险学员清单，只返回教师负责范围内的数据。

### 机构负责人

机构负责人用于机构管理视角，默认绑定：

```text
安心护理培训中心
```

机构负责人登录后，高风险学员清单只展示本机构学员。

## 密码配置

默认密码适合本地演示。部署时可以通过环境变量覆盖：

```text
ADMIN_PASSWORD
TEACHER_PASSWORD
ORG_PASSWORD
FLASK_SECRET_KEY
```

示例：

```bash
docker run --rm -p 5000:5000 \
  -e ADMIN_PASSWORD=your_admin_password \
  -e FLASK_SECRET_KEY=your_secret_key \
  smart-care-training
```
