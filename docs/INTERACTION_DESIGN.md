# 页面交互设计说明

本项目的前端不是单纯展示静态图表，而是围绕护理培训机构的日常管理场景设计交互。

## 使用角色

主要使用者包括：

- 培训机构管理员
- 护理课程教师
- 医疗/养老机构培训负责人

这些角色通常关心三个问题：

- 哪些课程培训效果不好？
- 哪些护理技能标签是薄弱环节？
- 哪些学员需要优先补训？

## 已实现交互

### 0. 工作台页签

页面拆分为四个页签：

- 总览
- 课程与薄弱项
- 风险跟踪
- 资源与趋势

这样可以避免所有图表和表格堆在一个长页面里。管理人员可以按工作任务切换视图，而不是一直向下滚动。

总览页增加“当前视角重点提醒”，用于把低分课程、高风险学员和资源优化建议前置到首屏，减少用户在多个图表之间来回寻找结论。

### 1. 课程类别筛选

可以按基础护理、临床技能、安全照护、康复护理等类别查看培训效果。

适用场景：

- 培训负责人想比较不同课程板块的整体效果。
- 教师想快速定位自己负责的课程类别。

### 2. 护理标签筛选

可以按失智照护、跌倒预防、压疮预防、慢病管理等标签筛选。

适用场景：

- 发现某类护理技能成绩偏低。
- 针对老年护理中的关键风险点做专项培训。

### 3. 风险阈值筛选

可以按风险值筛选高风险学员。

适用场景：

- 管理员只关注最需要干预的学员。
- 教师按优先级安排补训。

高风险学员清单采用固定高度滚动表格，表头固定，避免风险记录过多时把页面拉得很长。

### 4. 关键词搜索

支持搜索学员、机构、课程和护理标签。

适用场景：

- 某个机构负责人只看本机构学员。
- 教师查找某个具体学员的风险情况。

### 5. 图表点击联动

- 点击“标签薄弱环节”图表中的标签，会自动筛选该标签。

适用场景：

- 从宏观趋势图快速钻取到具体清单。
- 让看板具备“分析 -> 定位 -> 干预”的操作闭环。

### 6. 个性化建议联动

培训建议会根据当前筛选条件变化。

适用场景：

- 当筛选某个薄弱标签时，建议会偏向该标签相关课程。
- 当高风险学员较多时，建议优先安排分组补训。

## 设计取舍

早期版本提供了单课程筛选，但当用户选择单门课程后，课程柱状图只剩一个柱子，分析价值不高。因此当前版本去掉了单课程筛选，把具体课程查看放到“课程明细表”中，图表保留用于横向对比。

当课程类别或标签筛选后只剩 1 门课程时，系统不再展示单柱状图，而是切换为课程洞察卡片，展示课程名称、类别/标签、平均成绩、通过率、平均进度、学员数和累计学习分钟。这样更符合“单对象查看详情，多对象做对比”的产品原则。

本项目没有做复杂的登录、权限和课程编辑后台，原因是当前阶段重点是展示大数据分析链路：

```text
数据生成/采集 -> MySQL/HDFS 存储 -> Spark 分析 -> Flask API -> ECharts 交互展示
```

前端交互重点放在数据分析和培训干预决策上，避免把项目做成普通管理系统而削弱大数据主题。
