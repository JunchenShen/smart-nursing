# 页面交互设计说明

本项目的前端不是单纯展示静态图表，而是围绕护理培训机构的日常管理场景设计交互。

## 使用角色

主要使用者包括：

- 培训机构管理员
- 护理课程教师
- 医疗/养老机构培训负责人

这些角色通常关心三个问题：

- 哪些课程培训效果不好？
- 哪些护理技能标签是薄弱环节？
- 哪些学员需要优先补训？

## 已实现交互

### 1. 课程类别筛选

可以按基础护理、临床技能、安全照护、康复护理等类别查看培训效果。

适用场景：

- 培训负责人想比较不同课程板块的整体效果。
- 教师想快速定位自己负责的课程类别。

### 2. 护理标签筛选

可以按失智照护、跌倒预防、压疮预防、慢病管理等标签筛选。

适用场景：

- 发现某类护理技能成绩偏低。
- 针对老年护理中的关键风险点做专项培训。

### 3. 课程筛选

可以定位到某一门具体课程。

适用场景：

- 查看一门课程的平均成绩、通过率和相关风险学员。
- 课程优化前后做对比。

### 4. 风险阈值筛选

可以按风险值筛选高风险学员。

适用场景：

- 管理员只关注最需要干预的学员。
- 教师按优先级安排补训。

### 5. 关键词搜索

支持搜索学员、机构、课程和护理标签。

适用场景：

- 某个机构负责人只看本机构学员。
- 教师查找某个具体学员的风险情况。

### 6. 图表点击联动

- 点击“课程培训效果”图表中的课程，会自动筛选该课程。
- 点击“标签薄弱环节”图表中的标签，会自动筛选该标签。

适用场景：

- 从宏观趋势图快速钻取到具体清单。
- 让看板具备“分析 -> 定位 -> 干预”的操作闭环。

### 7. 个性化建议联动

培训建议会根据当前筛选条件变化。

适用场景：

- 当筛选某个薄弱标签时，建议会偏向该标签相关课程。
- 当高风险学员较多时，建议优先安排分组补训。

## 设计取舍

本项目没有做复杂的登录、权限和课程编辑后台，原因是当前阶段重点是展示大数据分析链路：

```text
数据生成/采集 -> MySQL/HDFS 存储 -> Spark 分析 -> Flask API -> ECharts 交互展示
```

前端交互重点放在数据分析和培训干预决策上，避免把项目做成普通管理系统而削弱大数据主题。
